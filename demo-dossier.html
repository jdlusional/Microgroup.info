Dossier

Dossier checks that a corporate foundation's own numbers add up: it reconciles what a foundation claims to give against what it actually filed, and cross-references a voluntary donation list against public 990 filings. This page runs Dossier against the same made-up foundation Atlas uses, so you can see how the two work together.

How to read this page. Thornbury Materials Foundation does not exist — the identical invented foundation used on the Atlas demo. Every claimed figure, filed figure, and donation-list row below is invented for this demo.

Unlike Almanac and Scribe's pattern (a fictional org checked against real funders), the reconciliation method and the 990 data structure are real; the foundation and every figure attached to it are invented. There is no real named foundation anywhere on this page.

Atlas ranks this foundation's giving against its peers. Dossier checks that the same foundation's own numbers add up.

How this was built, and what it cannot tell you

Where this comes from. Tier 1 sums Part XV grant rows from tx_990pf_grants_paid.csv for one filer_ein and tax_year. Tier 2 matches a voluntary donation list against public 990 and 990-PF filings by organization name, since the source corpus carries no recipient-side EIN.

What Dossier will not do. It never invents a claimed total when none is supplied. It never treats a name match as certain — every Tier 2 row carries a name-confidence note. It never computes a payout ratio or asset benchmark unless the filer's core financial fields are confirmed present in the corpus, which they are not for this family of foundations today.

What it cannot tell you. Whether an unmatched name reflects a real absence from IRS records or an unresolved name variant. Some real foundations file no Part XV detail at all, or sit outside the smaller Statistics-of-Income sample used for asset-based fields — both stated in general terms here since the demo entity is fictional.
