# microgroup.info

Enterprise/product site for MICRO Group, LLC — Jonathan Lin Davis's consulting business.
Static HTML/CSS/JS, no build tools or bundlers, hosted on Cloudflare Pages (auto-deploys on
push to `main`). Same conventions as the sibling jonathanlindavis.com repo (personal site):
branch-preview workflow, working files kept outside the repo, no model names in
client-facing copy.

Website Products (KCT, ACC), migrated here from jonathanlindavis.com — that repo now
301-redirects every old URL. The improvement log split by domain the same day: this repo's
`improvementlog.html` covers the Enterprise family (Almanac/Scribe/Concordance/Herald); the
Public Data family (GIFT/Interlock/F4F) keeps its own copy on
`jonathanlindavis.com/improvementlog.html`, cross-linked from this one. The Funding
Priorities Survey was retired the same day (`measure-survey.html` + its function removed,

owner instruction after its Cloudflare Access app briefly under-covered the `.html` URL (see
"Cloudflare Access gating" below for why, and the Websites Portfolio Context File for the
full account).

See the **Microgroup.info Project Context File** (on Google Drive at

## Backend functions (`functions/api/`)

  database, `contacts` table, `schema.sql`).
- `admin/contact-export.js` — `GET /api/admin/contact-export?key=...`, checked against
  `env.ADMIN_KEY`; exports the `contacts` table.
- `draft-request.js` — Scribe's "request a grant draft" form handler (Resend).
- `pricing-request.js` — `demo-pricing.html`'s request-pricing form handler (Resend,
  hard-fail).
- `grant-status/list.js` + `grant-status/update.js` — read/write the shared `grant-status`
  D1 database (binding `GRANT_STATUS_DB`, secret `GRANT_STATUS_KEY`), bound by reference to
  the SAME database as jonathanlindavis.com (same Cloudflare account, no separate copy) so
  status set from either domain is immediately visible from the other. Almanac's admin
  status UI (`measure-almanac.html` etc.) is the read/write client.

## Cloudflare Access gating

Per-path `self_hosted` Access apps, each referencing the SAME reusable account-level
policies already used on jonathanlindavis.com's equivalents (Allowed Testers, Owner,
Claude Code Access, plus each org's own staff policy) rather than duplicating policy
content:

| App | Path | Staff policy |
|---|---|---|
| MEASURE | `/measure*` | Measure Staff & Directors |
| Standpoint | *(none — genuinely ungated)* | — |
| Black Mamas Village | `/blackmamas*` | Black Mamas Village Staff & Directors |
| ACC | `/acc*` | ACC NPA Staff (`@austincc.edu`) |
| KCT Oppo | `/kct-oppo*` | Katelyn Thompson |

**Important, load-bearing:** this repo has no clean-URL middleware, so an exact-path Access
app domain (e.g. `/kct-oppo` with no wildcard) will NOT cover the `.html` variant of that
same URL — a real gap hit and fixed 2026-07-23. Any new exact-path gate on this domain
should use a wildcard (`/path*`) instead, or add the middleware first.

A separate, pre-existing wildcard app (`*.microgroup-info.pages.dev`, "Microgroup Previews")
blanket-gates every preview deployment regardless of which of the above apps would apply in
production — expected, not a bug, and not a substitute for the per-path production gates.

## Pattern-book governance

The Enterprise + Demo product families are governed by the SAME pattern book that governs
jonathanlindavis.com's own product pages — `conventions-products.html` and
`conventions-switcher.html`, generated by
`..\jonathanlindavis-pipelines\pattern-book\build_pattern_book.py`, **live on
jonathanlindavis.com, not in this repo** (that repo's own `roster.json` carries a `"repo"`
field per page, resolved against a `SITE_REPO_MICROGROUP` env var, so the generator reads
pages from either repo). Signature-component or shared-layer changes to any page in this
repo should still start on the owner page (or `tool-shared.css`, a physical copy kept in
sync with jonathanlindavis.com's own), then the book regenerated on the other repo — see
`conventions-products.html` there for the full governance model. `tool-shared.css` in this
repo is a copy, not a symlink; changes to the shared layer need to land in both repos'
copies.

## Conventions

- **Branch preview for visible changes:** push to a branch, share the Cloudflare Pages
  preview URL (`https://<branch>.microgroup-info.pages.dev`), then merge once approved.
  Small technical fixes can go straight to `main`.
- **Working files stay out of the repo:** anything not meant to ship lives in a sibling
  `<Project> subfolder (files not website)` folder on the Drive, never committed here —
  same convention as jonathanlindavis.com.
- **No model names in client-facing copy:** drafts are described as "produced with a
  combination of Claude Code, Python, CSS, and other programming tools," never naming the
  specific model.

Full detail on all of the above in the **Microgroup.info Project Context File** and the
**Websites Portfolio Context File**.
